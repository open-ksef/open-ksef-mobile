# Security Policy

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| latest  | Yes                |

## Reporting a Vulnerability

**Please do NOT open a public GitHub issue for security vulnerabilities.**

If you discover a security vulnerability in OpenKSeF Mobile, please report it responsibly:

1. **Email:** Send a description to **openksef@proton.me**
2. **Include:**
   - A description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

We will acknowledge your report within **48 hours** and aim to provide a fix or mitigation within **7 days** for critical issues.

## Scope

This policy applies to the OpenKSeF Mobile codebase (Android and iOS apps). It does not cover:

- Third-party dependencies (report to their respective maintainers)
- The KSeF test environment operated by the Polish Ministry of Finance
- The backend API (see the [open-ksef](https://github.com/OpenKSeF/open-ksef) repository)

## Security Best Practices for Users

- Download the app only from official distribution channels
- Keep the app updated to the latest version
- Use a strong Keycloak password for your account
- Rotate KSeF API tokens periodically
- Do not share your KSeF token or app credentials

## Acknowledgments

We appreciate the security research community and will credit reporters (with permission) in release notes.
